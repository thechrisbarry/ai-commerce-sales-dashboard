🔄 Data Flow Architecture

Feed = Booked Opportunities (not raw calls). Each row in the Live Feed is a unique prospect who was booked as an opportunity in Close.com. Deduplicated by lead name. Enriched with LeadFi data + call duration from the longest call to that lead. Paginated up to 500 records per request.

🔍 Source Parsing Logic

How Source Attribution Works

Each lead's "Pre-filled Calendly" custom field in Close.com contains a URL with tracking parameters. The API parses these in order:

1. sl= param → fb=Facebook, ig=Instagram, tt=TikTok, gg=Google, yt=YouTube, em=Email, org=Organic

2. utm_source= param → direct value

3. fbclid in URL → "Meta"

4. gclid in URL → "Google"

5. ttclid in URL → "TikTok"

6. Campaign from utm_campaign param

Example: ?sl=ig&fbclid=IwZXh0... → Source: "Instagram"
